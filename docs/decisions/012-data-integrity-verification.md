# ADR-012: 데이터 무결성 자동 검증 시스템

- 날짜: 2026-03-19
- 상태: 채택됨
- 결정: `integrity_check.py`로 6항목(DB 연결, 엔티티 수, 전이 일관성, 파일 동기화, 인덱스 정합, 스키마 버전)을 자동 검증한다. T9 Dashboard에 `/integrity` 페이지를 추가하여 결과를 실시간 시각화한다.
- 이유:
  - T9 OS는 SQLite + 마크다운 + git의 혼합 시스템이다. 데이터 불일치가 조용히 발생할 수 있다(고스트 엔티티, 파일 없는 레코드, 깨진 전이 체인 등).
  - 수동 확인은 비현실적이다. 엔티티 274건+, 파일 수천 개에서 불일치를 사람이 찾을 수 없다.
  - 대시보드 시각화는 "문제 있음"을 한눈에 보여주어 조기 대응을 가능하게 한다.
  - 감시단 G3(규칙)이 검증 결과를 참조하여 시스템 건강도를 판단할 수 있다.
- 대안:
  - **수동 SQL 쿼리**: 반복 불가, 기준 불명확 — 폐기.
  - **외부 모니터링 도구**: T9 OS 구조에 특화된 검증 불가 — 불채택.
  - **검증 없이 운영**: 불일치 누적 시 신뢰도 붕괴 — 폐기.
- 결과:
  - `pipes/integrity_check.py` 구현 완료. 6항목 자동 검증, JSON 출력.
  - T9D `/integrity` 페이지에서 pass/fail/warn 상태를 시각화.
  - 새로고침 버튼에 무결성 검증 통합(ADR-017 참조).

## Simondon Mapping
이 결정이 시몽동의 어떤 원리를 구현하는가: 시스템의 메타안정성을 정량적으로 측정 — 개체화된 시스템이 안정 상태를 유지하고 있는지를 수치로 표현하여, 메타안정성이 깨지기 전에 감지한다.
