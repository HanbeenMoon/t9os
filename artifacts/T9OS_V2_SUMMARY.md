# 10분 안에 전부 파악 가능

## 한 줄: 뭐가 바뀌는 거야?

> cc에게 "이거 하지마" 말로 하던 것 → **코드로 물리적으로 막는 것**으로 전환.

---

## 왜?

cc에게 규칙 54개 줬는데 절반밖에 안 지킴 (학술 연구 결과 48%).
**프롬프트의 문제가 아니라 구조의 문제.** 외부에서 강제하면 93%까지 올라감.

한빈이 AT1에서 발견한 것과 같은 원리:
> "인간의 판단이 들어가야 한다는 것 자체가 사고의 한계"
> 규칙 54개 = 과도한 규제 = 시장 실패

---

## 뭐가 바뀌는지 — 그림으로

### 지금 (v1)
```
한빈 → cc(혼자 전부) → 결과
          ↑ 규칙 54개를 "읽긴" 함
          ↑ 근데 안 지킴
```

### 앞으로 (v2)
```
한빈(설계자)
  ↓ 방향만 잡음
정책 엔진 (2층)
  ├─ 하드 게이트: 셸 스크립트 (rm -rf 같은 절대 차단 — 물리 법칙)
  └─ 소프트 게이트: 별도 LLM이 맥락 판단 (Build vs Buy — 살아있는 법률)
  ↓
cc(실행) → 검증(감시단 자동) → 결과
  ↑ 잘못하면 차단됨 + "왜" 피드백까지 받음
```

**핵심: 정책 엔진도 살아있다.** 삼성 법무팀처럼 사례가 쌓이면 판단이 진화한다.

## 핵심 변경 5가지

| # | 지금 | 앞으로 |
|---|------|--------|
| 1 | 규칙 54개 (말로, 죽어있음) | **2층 정책엔진** (하드=물리법칙 + 소프트=살아있는 LLM) |
| 2 | 감시단은 사후 리뷰 | **사전 차단** (커밋 전에 검증) |
| 3 | 전개체 247개 수동 분류 | **자동 분류** (Pipeline Composer) |
| 4 | CLAUDE.md 300줄 | **100줄** 이하 |
| 5 | MCP 95K 토큰 낭비 | **30K** 이하 |

## 삼성으로 비유하면

| 삼성 | T9 OS v2 |
|------|----------|
| 이재용 (방향 설정) | 한빈 |
| 물리적 보안 시스템 (출입 차단) | 하드 게이트 (rm -rf 같은 절대 차단) |
| 법무팀 (사례별 판단, 진화하는 규정) | **소프트 게이트 (LLM이 맥락 판단, 살아있음)** |
| 실무팀 (제품 개발) | cc + cx + gm |
| 감사팀 (품질 검증) | 감시단 (G1~G7) |

법무팀은 사람이라 살아있잖아. 판례가 쌓이고, 새 사례에 맞게 해석이 바뀌고.
**소프트 게이트도 LLM이라 살아있어.** 과거 피드백이 쌓이면 판단이 진화해.
