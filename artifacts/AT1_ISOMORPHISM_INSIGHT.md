## 핵심 인사이트: 예선 성공의 동형성 분석

한빈의 AT1 예선 30분 완풀은 **정규화 이론으로 설명**된다.

### 과적합-정규화 프레임

| | 다른 참가자 | 한빈 |
|--|-----------|------|
| **사전지식** | 시스템 설계, 프롬프트 엔지니어링, 도구 5개+ | "모른다" (무정보 사전확률) |
| **ML 용어** | 강한 prior (과적합 위험) | uninformative prior (드롭아웃 효과) |
| **경제학 용어** | 과잉 규제 (시장 경직) | 자유 시장 (AI 자율성 신뢰) |
| **결과** | 시스템 장애, 시간 초과 | 30분 완풀 |

### 왜 "모르는 것"이 무기가 되는가

```
참가자 A의 접근:
  인간이 문제 해석 → 분류 → AI에 분배 → 시스템으로 검증
  = min L(θ) + λ_strong · R(θ)    # 과도한 정규화 → 과소적합 아닌, 시스템 복잡도가 새로운 과적합

한빈의 접근:
  문제 원문 → cc 직접 투입 → 결과 수령
  = min L(θ)                      # 제약 없는 최적화 → AI의 표현력 최대 활용
```

핵심: **인간의 개입 자체가 정규화항**이다.
- 적절한 개입(λ_optimal) = 질문 리프레이밍, 결과 검증
- 과도한 개입(λ_large) = 문제 해석 대행, 도구 설계, 시스템 설정
- 한빈은 직관적으로 λ ≈ 0에서 시작 → 필요할 때만 개입 = **적응적 정규화**

### 제1원칙사고와의 연결

```
숙련도 곡선:
  낮음 → 단순 (과소적합: 문제를 이해 못함)
  중간 → 복잡 (과적합: 시스템/규칙이 너무 많음)
  최고 → 단순 (최적 정규화: 정확히 필요한 만큼만)
```

한빈은 "배운 적 없음" → 중간 단계를 건너뛰고 → 최고 숙련도의 단순함에 도달.
이것이 "언러닝이 아니라 비학습(non-learning)"의 힘이다.

## 본선 전략에 반영

### Phase 0 (0~5분): 전 문제 스캔
- **적용**: 문제를 인간이 해석하지 마라. cc에게 원문 전달, cc가 난이도/유형 판단.
- **동형성**: uninformative prior 유지. 인간의 편견이 난이도 판단을 왜곡할 수 있음.

### Phase 1 (5~120분): 병렬 풀이
- **적용**: 각 세션에 최소한의 지시만. "이 문제를 풀어라" — 시스템 설계 X, 템플릿 X.
- **동형성**: 드롭아웃 효과 — 각 세션이 독립적으로 다른 경로 탐색.

### Phase 2 (120~150분): 교차 검증
- **적용**: 이때 비로소 인간 개입 = 적응적 정규화. 답안 간 불일치 발견 시만 개입.
- **동형성**: 앙상블 방법 — 여러 모델의 합의가 단일 모델보다 강건.

### Phase 3 (150~180분): Hard 집중
- **적용**: Hard 문제에서만 인간 판단 투입 = 정규화 강도 증가.
- **동형성**: early stopping 대신 추가 학습 — 쉬운 문제에서 절약한 시간을 투자.
