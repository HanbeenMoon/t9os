# ADR-040: 레거시 데이터 다이제스트 파이프라인 — 옵시디언+카톡+노션

- 날짜: 2026-03-15 (다이제스트 완료)
- 상태: 채택됨
- 결정: 개인 데이터(옵시디언 1,918개, 카카오톡 16개, 노션 DB 9개)를 다층 다이제스트로 압축한다. 팩트 다이제스트(*_digest.txt), 사고 지도(THOUGHT_MAP.md, 7테마), 원시 인사이트(RAW_INSIGHTS.md, 236개)를 생성하고, 접근 순서를 사고지도→팩트→인사이트→grep→원본으로 정한다.
- 이유: 원본 2,421개를 매번 전수 읽기는 토큰 낭비. 계층적 압축으로 대부분의 질문에 다이제스트만으로 답변 가능하게 하되, 원본 접근 경로를 보장.
- 대안: 원본 전수 검색만 (토큰 과소비), 벡터 DB 구축 (Build vs Buy 위반, grep이 충분), 요약 없이 FTS만 (맥락 파악 불가)
- 결과: 총 169건 다이제스트, 343건 FTS 인덱싱. "인덱스 시스템 별도 Build 금지 (grep이 Buy)" 규칙 확립
- 출처: 20260316_CC_003_041500_T9OS_v02_시몽동개정_model-project_project-alpha.txt, CLAUDE.md 데이터 접근 순서 섹션

## Simondon Mapping
구체화(concretization)의 계층화: 원본(전개체)→다이제스트(중간체)→인사이트(안정체)로 정보의 개체화 수준을 계층적으로 관리.
