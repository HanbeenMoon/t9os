# ADR-033: UX Five Principles — Zero Decision Design

- Date: 2026-03-16
- Status: Accepted
- Decision: Establish 5 UX principles: (1) Zero Decision, (2) One Breath, (3) Peripheral Awareness, (4) Error as Conversation, (5) Progressive Disclosure. Introduce shortcut commands and CLI alias.
- Rationale: Research-backed — Zero UI reduces task completion time by 68%, 71% of users expect context-adaptive behavior, solution-including error messages reduce abandonment by 40%.
- Outcome: `telos/UX_PRINCIPLES.md`, CLI shortcuts (`done`, `go`, `do`)

## Simondon Mapping
Zero Decision mirrors the preindividual's resistance to premature classification — the system should not impose form (choices) on the user.
