# ADR-017: 새로고침 버튼에 무결성 검증 통합

- 날짜: 2026-03-19
- 상태: 채택됨
- 결정: T9 Dashboard의 새로고침 버튼 클릭 시 데이터 갱신과 무결성 검증을 동시에 수행한다. 관찰(데이터 조회)과 검증(데이터 정합성 확인)을 분리하지 않는다.
- 이유:
  - 새로고침은 "최신 상태를 보겠다"는 의도다. 최신 상태가 정합적인지 동시에 확인하는 것이 자연스럽다.
  - 별도 "검증" 버튼을 두면 사용자(한빈)가 별도로 눌러야 한다 — 안 누른다.
  - integrity_check.py 실행은 1초 미만이므로 새로고침 UX에 영향 없다.
  - 문제 발견 시 대시보드에 즉시 경고를 표시하여 조기 대응한다.
- 대안:
  - **별도 검증 페이지만**: 접근 빈도 낮음, 문제 발견 지연 — 폐기.
  - **주기적 자동 검증**: 서버 필요, Vercel 정적 배포와 맞지 않음 — 불채택.
  - **검증 없이 표시만**: 잘못된 데이터를 보여주는 것이 더 위험 — 폐기.
- 결과:
  - 새로고침 API가 `export-t9data.sh` + `integrity_check.py`를 순차 호출.
  - 무결성 결과를 대시보드 상단에 요약 표시(pass 수/전체, 경고 있으면 배지).
  - `/integrity` 페이지에서 상세 결과 확인 가능.

## Simondon Mapping
이 결정이 시몽동의 어떤 원리를 구현하는가: 관찰과 검증을 분리하지 않음 (전도적 관찰) — 시몽동의 전도(transduction)에서 정보 전달은 동시에 구조를 검증한다. 데이터를 보는 행위 자체가 데이터의 정합성을 확인하는 행위와 동일하다.
