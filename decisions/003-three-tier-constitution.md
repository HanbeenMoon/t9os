# ADR-003: Three-Tier Constitution (L1/L2/L3)

- Date: 2026-03-15
- Status: Accepted
- Decision: Separate all T9 OS rules into three tiers:
  - **L1 (Execution)**: Operational invariants. Agent roles, file rules, token rules, SRBB, guardian obligations, autonomous work principle.
  - **L2 (Interpretation)**: State transition criteria. Disparation-based transitions, five-axis interpretation, preindividual triage, guardian interpretation.
  - **L3 (Amendment)**: Meta-rules for modifying L1 and L2. Amendment triggers, procedures, concretization measurement, modulation principle, non-retroactivity.
- Rationale:
  - A flat rule list creates contradictions and lacks hierarchy. "Which rule wins?" has no answer.
  - Legal theory insight (Kelsen): rules need hierarchy. Constitutional rules outrank procedural rules outrank operational rules.
  - Self-amendment capability prevents ossification — the system can rewrite its own rules through a governed process.
- Outcome: `constitution/L1_execution.md`, `L2_interpretation.md`, `L3_amendment.md`

## Simondon Mapping
Modulation — the constitution itself is an object of continuous, variable self-formation, not a fixed template.
