# ADR-003: 3층 헌법 구조 (L1/L2/L3)

- 날짜: 2026-03-15 (v0.1), 2026-03-16 (v0.2 시몽동 전면 개정)
- 상태: 채택됨
- 결정: T9 OS의 모든 규칙을 3개 계층으로 분리한다.
  - L1 (실행 규칙): 지금 당장 돌아가게 하는 규칙. 위반 불가. 에이전트 역할, 파일 규칙, 토큰 규칙, SRBB, 감시단 의무, 자율 연속 작업 원칙 등.
  - L2 (해석 규칙): 상태 전이 판단 기준. 이접 기반 전이 조건, 5축 해석, 전개체 정리 프로토콜, 감시단 해석 기준 등.
  - L3 (개정 규칙): 1층과 2층을 언제 수정하는가. 개정 트리거, 절차, 구체화 수준 측정, 변조 원칙.
- 이유:
  - 웹 Claude 시절 "cc 자체 판단으로 규칙을 바꾸면 세션 간 일관성이 깨지는" 문제가 있었다.
  - 메타 무한회귀 해결: "규칙을 만드는 규칙을 만드는 규칙..."을 3층에서 끊는다.
  - 시몽동의 변조(modulation) 원칙 적용: "완결된 OS가 아니라, 자기 자신의 규칙을 언제든 수정할 수 있는 가설적 OS." L3 자체도 개정 대상이다.
  - 실행(what to do)과 해석(when to transition)과 개정(when to change rules)을 분리해야 각 계층을 독립적으로 변조할 수 있다.
- 대안:
  - **단일 규칙 문서**: 규칙이 커지면 모순 발생, 개정 범위 파악 불가 — 폐기.
  - **4층 이상**: 과도한 메타 작업, 규칙 자체가 목적이 되는 함정 — 불채택.
  - **규칙 없음 (자율 판단)**: 세션 간 일관성 보장 불가 — 폐기.
- 결과:
  - `constitution/L1_execution.md`, `L2_interpretation.md`, `L3_amendment.md`에 각각 구현.
  - L1/L2 변경 시 반드시 CLAUDE.md + memory 동시 갱신 (3곳 동시 갱신 원칙).
  - L3 개정 로그에 모든 변경 이력이 기록된다.
  - 한빈 승인 없이 L1/L2 변경 금지 (L3 규칙).
  - 개정 빈도가 주 3회 이상이면 L3 자체를 점검하는 메타 규칙이 존재한다.

## Simondon Mapping
이 결정이 시몽동의 어떤 원리를 구현하는가: 변조(modulation) — 완결된 형태(form)가 아니라 자기 규칙을 지속적으로 수정하는 가설적 체계를 3층 구조로 보장한다.
