# ADR-016: 침전 엔티티 재활성화 (resurface) 기능

- 날짜: 2026-03-19
- 상태: 채택됨
- 결정: `t9_seed.py resurface` 명령으로 침전(sediment) 상태 엔티티를 다시 전개체(preindividual)로 끌어올리는 기능을 추가한다. 침전은 삭제가 아니라 잠재성의 가라앉음이며, 재활성화 경로가 반드시 존재해야 한다.
- 이유:
  - 시몽동 철학에서 침전은 죽음이 아니다. 조건이 변하면 잠재성이 다시 활성화될 수 있다.
  - 현재 `transition → sediment`는 일방향이다. 한번 침전하면 DB에 남지만 접근 경로가 없다.
  - 프로젝트 방향 전환, 새로운 맥락 발생 시 과거 침전 엔티티가 다시 유용해질 수 있다.
  - `consolidate` 명령이 아카이브 → memory 통합을 하지만, 침전 → 전개체 복원은 별개의 기능이다.
- 대안:
  - **침전 영구 삭제**: 시몽동 위반. 잠재성 소멸 — 폐기.
  - **수동 SQL UPDATE**: 재현성 없음, 전이 이력 누락 — 폐기.
  - **자동 resurface (일정 기간 후)**: 노이즈 과다, 의도적 resurface만 의미 있음 — 불채택.
- 결과:
  - `t9_seed.py resurface <id> [사유]`: sediment → preindividual 전이, 이력 기록.
  - `t9_seed.py search` 결과에 침전 엔티티도 포함(표시 구분).
  - daily 브리프에 "오래된 침전 중 재검토 후보" 제안 기능 추가 검토(향후).

## Simondon Mapping
이 결정이 시몽동의 어떤 원리를 구현하는가: 침전은 죽음이 아닌 잠재성의 가라앉음. 재활성화 경로 필수 — 개체화가 완료되어 더 이상 활성이 아닌 것(침전)도 새로운 긴장이 발생하면 다시 전개체 상태로 돌아갈 수 있다. 이 경로가 없으면 시스템은 엔트로피 증가(정보 사망)로 향한다.
