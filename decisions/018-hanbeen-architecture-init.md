# ADR-018: workspace 아키텍처 초기화 — CLAUDE.md 에이전트 컨텍스트 시스템

- 날짜: 2026-03-04
- 상태: 채택됨
- 결정: workspace 레포의 루트에 `CLAUDE.md`와 `AGENTS.md`를 배치하여 AI 에이전트(cc, cx)가 세션 시작 시 자동으로 프로젝트 컨텍스트를 로드하게 한다. 이후(03-10) CLAUDE.md를 대폭 확장하여 환경, 폴더 구조, 역할 분담, 로깅 규칙, 프로젝트 목록 등 모든 오케스트레이션 규칙을 단일 파일에 집약한다.
- 이유:
  - AI 에이전트는 세션 간 기억이 없다. 매 세션 시작 시 동일한 컨텍스트를 주입해야 일관성이 유지된다.
  - Claude Code는 `CLAUDE.md`를 자동으로 읽는다. 이 메커니즘을 활용하면 수동 프롬프트 없이 컨텍스트가 로드된다.
  - 분산된 설정 파일 대신 단일 파일에 집약하면 에이전트가 참조해야 할 파일 수가 줄고, 토큰 효율이 높아진다.
  - Codex도 `AGENTS.md`를 자동으로 읽으므로 동일 패턴을 적용한다.
- 대안:
  - **매 세션 수동 프롬프트**: 반복 노동, 누락 위험, 컨텍스트 불일치 — 폐기.
  - **별도 config 파일 (YAML/JSON)**: 에이전트가 자동으로 읽지 않음, 별도 파싱 필요 — 불채택.
  - **메모리 시스템만 의존**: memory는 축적형이지 구조화된 규칙 전달에 부적합 — 보완재로만 사용.
- 결과:
  - `CLAUDE.md`가 T9 OS의 "운영체제 부트로더" 역할을 한다.
  - 모든 cc 세션은 CLAUDE.md → WORKING.md → state.md 순서로 컨텍스트를 로드한다.
  - 프로젝트 규칙 변경 시 CLAUDE.md만 갱신하면 모든 세션에 즉시 반영된다.

## Simondon Mapping
이 결정이 시몽동의 어떤 원리를 구현하는가: 연합된 환경(milieu associé)의 초기 씨앗 — CLAUDE.md가 에이전트의 개체화를 위한 초기 조건(정보적 환경)을 제공하여, 매 세션이 동일한 메타안정 상태에서 출발하게 한다.
