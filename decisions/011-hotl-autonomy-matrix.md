# ADR-011: HITL에서 HOTL로 전환 — 자율성 티어 도입

- 날짜: 2026-03-19
- 상태: 채택됨
- 결정: Human-in-the-Loop(HITL)에서 Human-on-the-Loop(HOTL)로 전환한다. 3단계 자율성 티어를 도입한다: T1(완전자율, 승인 불요), T2(사후보고, 실행 후 알림), T3(사전승인, 설계자 확인 필수). L1에 티어별 작업 분류를 명시한다.
- 이유:
  - 기존 HITL 모델은 모든 결정에 설계자의 개입을 요구하여 병목이 발생한다. 특히 야간/장기 세션에서 cc가 대기 상태로 토큰을 낭비한다.
  - T1(파일 정리, 로그, 감시단 실행 등)은 설계자에게 묻는 것 자체가 비효율이다.
  - T2(코드 생성, 문서 수정 등)는 실행 후 결과를 보고하면 충분하다.
  - T3(L1/L2 변경, 금전 관련, 외부 API 키 사용 등)만 사전 승인이 필요하다.
  - 이미 다수의 memory 피드백("묻지 말고 진행", "trojan 전임 자율 실행")이 이 방향을 가리킨다.
- 대안:
  - **전면 자율**: 실수 시 되돌리기 어려운 작업(L1 변경, 금전)에서 위험 — 폐기.
  - **기존 HITL 유지**: 병목 지속, 설계자 에너지 낭비 — 폐기.
  - **작업별 개별 설정**: 관리 복잡도 과다, 3티어가 충분히 커버 — 불채택.
- 결과:
  - L1에 T1/T2/T3 티어 정의와 작업 분류표 추가.
  - cc는 작업 시작 시 티어를 판단하고, T3만 설계자에게 확인한다.
  - 티어 오분류 시 설계자이 교정하면 L1 분류표를 즉시 갱신한다.

## Simondon Mapping
이 결정이 시몽동의 어떤 원리를 구현하는가: 개체의 자율적 개체화를 허용하되 메타안정성 범위 내에서 — 시스템(cc)이 스스로 개체화할 수 있는 영역(T1/T2)을 명시하고, 메타안정성을 깨뜨릴 수 있는 영역(T3)에서만 외부 에너지(설계자 승인)를 요구한다.
