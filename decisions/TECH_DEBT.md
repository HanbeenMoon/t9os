# T9 OS 기술 부채 목록

## 심각도: HIGH (가동에 영향)

### TD-001: t9_bot.py systemd 미연결
- **위치**: `pipes/t9_bot.py`
- **현상**: 코드는 작성되어 있으나 systemd에 등록되지 않아 가동되지 않는다.
- **영향**: 모바일/야외에서 cc 접근 불가. 텔레그램 봇이 죽어 있다.
- **해결**: systemd unit 파일 작성 + 등록 + 자동 재시작 설정.

### TD-002: deadline_notify.py cron 미연결
- **위치**: `pipes/deadline_notify.py`
- **현상**: 마감일 알림 코드는 있으나 cron에 등록되지 않아 알림이 안 간다.
- **영향**: 마감일을 놓칠 수 있다. daily에서만 확인 가능.
- **해결**: crontab 등록.

### TD-003: 마감일 파일 경로 불안정
- **위치**: `t9_seed.py` L27-32 (`DEADLINE_CANDIDATES`)
- **현상**: 마감일 파일 경로가 4개 후보를 fallback으로 탐색한다. 심볼릭 링크 깨짐 가능성.
- **영향**: 마감일 데이터 누락 시 daily 브리프에서 마감 정보가 빠진다.
- **해결**: 정식 경로 하나를 확정하고 나머지는 제거.

## 심각도: MEDIUM (기능 제한)

### TD-004: Pipeline Composer 미구현
- **위치**: BIBLE.md 섹션 15 핵심 갭 #1
- **현상**: T9 OS의 핵심 컨셉인 "파이프라인을 만드는 파이프라인"이 미구현이다.
- **영향**: 새 파이프라인은 수동으로 코드를 작성해야 한다. 자동 워크플로우 생성 불가.
- **해결**: compose 명령 확장 또는 별도 Composer 모듈 설계.

### TD-005: Resource 추적 시스템 미구현
- **위치**: BIBLE.md 섹션 3.4 (5축 중 자원축)
- **현상**: 5축 해석 중 자원(Resource) 축이 완전히 미구현이다. 시간, 토큰, 돈 추적이 안 된다.
- **영향**: 토큰 사용량, 작업 시간 등의 데이터 기반 의사결정 불가.
- **해결**: 세션별 토큰 사용 로깅 + 시간 추적 최소 구현.

### TD-006: 심리학 7모듈 대부분 미구현
- **위치**: BIBLE.md 섹션 9
- **현상**: 7개 모듈 중 reflect(반성 모듈)만 부분 구현. 나머지 6개(동기, 감정번역, 행동전환, 심리적 유연성, Flow 교정, Allostatic 예측) 미구현.
- **영향**: 심리적 상태 고려 없이 작업 배분. 번아웃 예측 불가.
- **해결**: Phase 1 구현 대상 선정이 필요 (BIBLE TODO 항목).

### TD-007: 자동 split/merge 판단 미구현
- **위치**: BIBLE.md Layer B 설명
- **현상**: split/merged 전이는 수동으로만 가능하다. cc가 자동으로 "이건 쪼개야 한다" 판단을 하지 못한다.
- **영향**: 엔티티가 비대해지거나, 유사 엔티티가 분산된 상태로 방치된다.
- **해결**: 패턴 기반 자동 제안 (확정은 한빈 승인).

### TD-008: 오케스트레이터 P7 (cc/cx/gm 자동 선택) 미구현
- **위치**: BIBLE.md 섹션 10, L2U P7
- **현상**: 어떤 에이전트가 적합한지 수동 판단해야 한다. `routing-policy.md`가 가이드 역할만 한다.
- **영향**: cc가 직접 해야 할 작업을 cx/gm에 위임하지 못하거나, 그 반대가 발생.
- **해결**: 작업 유형별 자동 라우팅 규칙 구현.

### TD-009: 오버나이트 시스템 미구현
- **위치**: BIBLE.md 섹션 14 Buy 로드맵
- **현상**: cron + session-brief.md 기반 오버나이트 자동 작업이 구현되지 않았다.
- **영향**: 야간에 자동으로 reindex, 정리, 브리프 생성 등을 수행할 수 없다.
- **해결**: cron job 등록 + 브리프 자동 생성 스크립트.

## 심각도: LOW (개선 사항)

### TD-010: bare except 남용
- **위치**: `t9_seed.py` 여러 곳 (L155, L253, L486 등)
- **현상**: `except:` (bare except)로 모든 예외를 무시하는 코드가 여러 곳에 있다.
- **영향**: 실제 에러가 숨겨져서 디버깅이 어렵다.
- **해결**: `except Exception as e:` + 로깅으로 변경.

### TD-011: WSL-NTFS 크로스 파일시스템 workaround
- **위치**: `t9_seed.py` L426-431 (`cmd_transition`)
- **현상**: `shutil.move`가 WSL->NTFS에서 실패할 수 있어 copy+delete fallback이 있다. 원본 삭제 실패 시 DB만 업데이트되고 파일은 이중으로 존재.
- **영향**: 드물지만 파일 이중 존재 가능.
- **해결**: 이중 파일 감지 + 정리 루틴 추가.

### TD-012: FTS5 인덱스 재구축 로직 없음
- **위치**: `t9_seed.py` `_upsert()`, `cmd_reindex()`
- **현상**: FTS5 가상 테이블 생성 실패 시 `try/except`로 무시한다. 인덱스가 깨져도 복구 수단이 없다.
- **영향**: 검색 결과 누락 가능. fallback으로 LIKE 검색이 작동하지만 느리다.
- **해결**: FTS5 인덱스 재구축 명령 추가 (rebuild).

### TD-013: Concretization 수준 측정 미구현
- **위치**: `constitution/L3_amendment.md`, `telos/SIMONDON.md`
- **현상**: L3에 구체화 수준 4단계(추상/연결/통합/구체)가 정의되어 있으나 자동 측정이 없다.
- **영향**: T9 OS의 진화 방향을 데이터로 판단하지 못하고 감으로 판단.
- **해결**: 도구 간 통합 지표 정량화 (자동 호출 횟수, 수동 개입 비율 등).

### TD-014: Transduction 경로 추적 부실
- **위치**: `t9_seed.py` `cmd_relate()`, `relates` 테이블
- **현상**: relates 테이블에 direction과 description이 있지만, "A의 패턴이 B의 원리가 됨" 같은 전도적 학습 경로를 체계적으로 추적하지 못한다.
- **영향**: 프로젝트 간 패턴 전파 이력을 소급 추적하기 어렵다.
- **해결**: relate 시 전도 유형 분류 + 시각화.

### TD-015: calendar_sync.py 수동 실행
- **위치**: `pipes/calendar_sync.py`
- **현상**: Google Calendar 동기화가 수동으로만 실행된다.
- **영향**: 일정이 입력되어도 캘린더에 자동 반영되지 않는다 (memory에 "묻지 말고 바로" 규칙이 있으나 실제로는 수동).
- **해결**: 마감일 변경 감지 시 자동 동기화 트리거.

## 아키텍처 부채

### TD-A1: BIBLE.md와 실제 구현 간 GAP 관리
- 이전에 `BIBLE_IMPLEMENTATION_GAP.md`로 갭을 추적했으나, 이를 BIBLE v0.3에 인라인으로 병합했다. 갭이 다시 벌어지면 추적 수단이 없다.
- **제안**: ADR 시스템이 이 역할을 부분적으로 대체. 주기적 갭 점검 필요.

### TD-A2: 단일 DB 파일의 Syncthing 동기화 위험
- `.t9.db`가 SQLite WAL 모드로 동작하지만, Syncthing이 두 PC에서 동시에 DB를 수정하면 충돌 가능.
- 현재는 한빈이 주로 서울PC에서 작업하므로 문제가 안 되지만, 세종PC에서도 병렬 작업 시 위험.
- **제안**: Syncthing conflict 감지 + DB 머지 도구, 또는 DB 접근을 한 PC로 제한.

### TD-A3: 파이프라인 테스트 부재
- 14개 파이프라인 중 자동화된 테스트가 하나도 없다. `reproducibility_check.py`는 레지스트리 등록 여부만 확인하고 실제 동작 테스트는 안 한다.
- **제안**: 핵심 파이프라인(gm_batch guardian, session_lock, t9_seed 핵심 명령)에 최소 smoke test 추가.
