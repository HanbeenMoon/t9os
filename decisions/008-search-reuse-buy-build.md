# ADR-008: SRBB Principle (Search → Reuse → Buy → Build)

- Date: 2026-03-15
- Status: Accepted
- Decision: Before starting any new work, follow Search → Reuse → Buy → Build. Build is reserved exclusively for things that don't exist yet.
- Rationale: A 12-hour rabbit hole incident (Puppeteer → proxy → Zotero → SeleniumBase) could have been solved in 1 hour by searching for Bright Data first.
- Outcome: SRBB enforced as L1 rule, checked by G3 guardian, Orient layer blocks new pipe creation without SRBB confirmation.

## Simondon Mapping
Transduction — before creating new structure, check if existing structure can propagate to solve the problem.
