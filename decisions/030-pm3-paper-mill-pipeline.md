# ADR-030: PM3 학술 논문 자동화 파이프라인 아키텍처

- 날짜: 2026-03-01 (초기 설계), 2026-03-17 (아키텍처 문서화)
- 상태: 채택됨
- 결정: 학술 논문 생성을 모듈화된 파이프라인으로 구축한다. Module D(데이터 처리: D1 수집 → D3 전처리 → D5 조립), Module E(평가: 리뷰어 시뮬레이션)로 분리하고, pandoc + reference-doc 방식으로 DOCX를 생성한다.
- 이유: python-docx로 직접 표/서식을 코드 생성하면 서식이 깨지고 재현성이 없음(v6 실패). pandoc은 Markdown→DOCX 변환이 안정적이고, BibTeX/citeproc 자동 참고문헌 처리 가능.
- 대안: python-docx 직접 생성 (서식 깨짐), LaTeX (한글 지원 복잡), Google Docs API (오프라인 불가)
- 결과: SSK 논문 v7~v25까지 16회 수정 자동화, 55+ 리뷰어 시뮬레이션, PM3 퀵스타트 문서 생성
- 출처: 20260301_CC_002_2-t9_021430_PM3셋업.txt, 20260316_CC_001_015111_SSK논문재생성_T9OS정비.txt, 20260317_CC_004_220000_SSK_트목v2_PM3_전체완료.txt
