# ADR-061: No Edit Without Explicit Approval

- Date: 2026-03-17
- Status: Accepted
- Decision: The agent does not modify or create files without explicit designer approval. Conversation mode is default; execution mode requires directive. Exception: HOTL T1 (fully autonomous) tasks.
- Rationale: Proactive file modification without approval creates trust erosion. The designer must remain in control of what changes.
- Outcome: L1 rule, memory enforcement

## Simondon Mapping
The designer is the source of individuation direction — the system executes but does not autonomously determine form.
